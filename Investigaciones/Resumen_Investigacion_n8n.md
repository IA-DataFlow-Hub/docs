# 🤖 Informe de Investigación: Automatización con n8n

> [!abstract] **Resumen Ejecutivo**
> 
> El informe analiza a **n8n** como una solución de automatización de flujo de trabajo "Fair-Code" que permite conectar aplicaciones, bases de datos y APIs mediante una arquitectura basada en nodos. Es la pieza central para eliminar tareas manuales y fragmentación de datos.

---

## 🛠️ Puntos Clave de la Herramienta

## 1. Filosofía y Control

- **Auto-alojamiento (Self-hosting):** A diferencia de Zapier o Make, n8n permite instalarse en servidores propios. Esto garantiza el **control total de los datos** (privacidad) y costos predecibles.
    
- **Bajo Código (Low-Code):** Permite a usuarios técnicos y no técnicos crear flujos complejos sin escribir cientos de líneas de código.
    

## 2. Arquitectura de Funcionamiento

n8n opera bajo una lógica de **nodos**:

- 🔌 **Triggers (Disparadores):** Eventos que inician el flujo (ej: llega un correo, un cambio en la base de datos MySQL).
    
- ⚙️ **Nodos de Acción:** Transforman datos, realizan cálculos o llaman a servicios externos.
    
- 🔗 **Integración de APIs:** Capacidad nativa para conectar cualquier servicio que tenga una API REST.

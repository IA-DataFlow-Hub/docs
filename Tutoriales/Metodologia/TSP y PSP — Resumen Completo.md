# TSP y PSP — Resumen Completo

> Metodologías de ingeniería de software enfocadas en mejorar la calidad, productividad y organización del desarrollo.

---

# 📌 ¿Qué es PSP?

## PSP (Personal Software Process)

Es una metodología que ayuda a un desarrollador individual a trabajar de manera más organizada y eficiente.

Fue creada para que cada programador:

- mida su tiempo,
- detecte errores,
- estime mejor,
- escriba código de mayor calidad.

La idea principal es:

> “Si puedes medir cómo trabajas, puedes mejorar.”

---

# 📌 ¿Qué es TSP?

## TSP (Team Software Process)

Es la versión orientada a equipos.

Ayuda a que un grupo de desarrolladores trabaje de forma organizada mediante:

- roles definidos,
- planificación,
- métricas,
- seguimiento,
- calidad del software.

TSP usa las bases de PSP pero aplicadas al trabajo colaborativo.

---

# 🎯 Objetivos Principales

| PSP | TSP |
|---|---|
| Mejorar al desarrollador | Mejorar al equipo |
| Reducir errores | Coordinar trabajo |
| Estimar tiempos | Cumplir entregas |
| Aumentar calidad | Mejorar productividad |

---

# 🧠 Idea Central

Estas metodologías se basan en:

## ✔ Medición constante

Se registra:

- tiempo trabajado,
- cantidad de errores,
- líneas de código,
- productividad,
- retrabajo.

---

# 📊 Conceptos Importantes

## 1. Estimación

Predecir cuánto tiempo tomará una tarea.

Ejemplo:

```txt
Crear login → 5 horas
API usuarios → 3 horas
Tests → 2 horas
```

---

## 2. Métricas

Datos que permiten medir desempeño.

Ejemplos:

- Bugs encontrados
- Tiempo invertido
- Tareas terminadas
- Líneas de código

---

## 3. Defectos

Errores encontrados en el software.

Ejemplo:

```txt
El login permite contraseña vacía.
```

---

## 4. Revisión

Inspección del código antes de entregar.

Objetivo:

- encontrar errores temprano,
- reducir bugs en producción.

---

# 🔄 Ciclo PSP

## Flujo básico

```txt
Planear
   ↓
Diseñar
   ↓
Programar
   ↓
Revisar
   ↓
Probar
   ↓
Medir resultados
```

# 👨‍💻 Ejemplo Cotidiano (PSP)

## Situación

Juan desarrolla una API en Laravel.

Antes trabajaba así:

- programa sin medir tiempo,
- corrige bugs al final,
- no sabe cuánto demora realmente.

---

## Aplicando PSP

Ahora hace esto:

| Actividad | Tiempo |
|---|---|
| Diseño DB | 30 min |
| Backend | 2 h |
| Testing | 45 min |
| Corrección bugs | 20 min |

También registra:

```txt
Errores encontrados: 5
Errores evitados antes de producción: 4
```

---

## Resultado

Después de varios proyectos:

- estima mejor,
- reduce errores,
- trabaja más rápido,
- conoce sus puntos débiles.

---

# 👥 Ejemplo Cotidiano (TSP)

## Equipo Laravel + React

Antes:

- nadie sabe qué hace el otro,
- tareas duplicadas,
- atrasos constantes.

---

## Aplicando TSP

Se definen roles:

| Rol | Responsabilidad |
|---|---|
| Líder | Coordinar |
| Backend | APIs |
| Frontend | UI |
| QA | Testing |
| DevOps | Deploy |

Se hacen métricas semanales:

```txt
Tareas completadas: 24
Bugs encontrados: 8
Tiempo promedio por HU: 5 horas
```

---

## Resultado

- mejor organización,
- menos retrabajo,
- entregas más estables.

# 📈 Beneficios

## PSP

- Mejor disciplina
- Mejor estimación
- Menos bugs
- Más productividad

---

## TSP

- Equipos organizados
- Mejor comunicación
- Mayor calidad
- Proyectos más predecibles

---

# ⚠ Desventajas

| Problema | Explicación |
|---|---|
| Mucha documentación | Puede sentirse pesado |
| Requiere disciplina | No funciona si nadie mide |
| Más tiempo inicial | La mejora se ve después |

---

# 🆚 PSP/TSP vs Agile

| PSP/TSP | Scrum/Agile |
|---|---|
| Muy basado en métricas | Más flexible |
| Enfocado en calidad medible | Enfocado en iteraciones |
| Más disciplinado | Más adaptable |
| Muy formal | Menos rígido |

---

# 🧩 Relación con Scrum

PSP y TSP pueden complementar Scrum.

Ejemplo:

- Scrum organiza sprints.
- PSP mejora productividad individual.
- TSP mejora organización técnica del equipo.

---

# 📚 Glosario

## Defecto
Error o bug en el software.

---

## Métrica
Dato usado para medir rendimiento o calidad.

---

## Estimación
Predicción del tiempo o esfuerzo necesario.

---

## Productividad
Cantidad de trabajo realizado en cierto tiempo.

---

## Revisión de código
Inspección manual del código para detectar errores.

---

## Calidad de software
Qué tan confiable, mantenible y estable es el sistema.

---

# 🧠 Idea Clave Final

## PSP

Ayuda a que el desarrollador mejore individualmente mediante medición y disciplina.

## TSP

Ayuda a que los equipos desarrollen software de forma organizada y predecible.

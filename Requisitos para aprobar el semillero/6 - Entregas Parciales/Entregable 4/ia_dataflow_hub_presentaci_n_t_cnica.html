01. chaos-generator

HU-015: Automatización de datasets sintéticos caóticos para el entrenamiento de modelos de IA.

¿Qué es chaos-generator?

Concepto Clave

Un robusto generador automatizado de datos sintéticos con imperfecciones lógicas del mundo real, diseñados para entrenar algoritmos en normalización y curaduría.

El "Lado Sucio"

Un modelo de IA de limpieza no puede entrenarse con estructuras ideales. Este software inyecta escenarios adversos para forzar el auto-reconocimiento y la resiliencia.

Clasificación de Anomalías de Datos

Categoría de Caos Problema Técnico que Representa
REDUNDANCIADuplicados idénticos y duplicación difusa (fuzzy match)
ESTRUCTURA_ROTACeldas truncadas, marcas BOM huérfanas, codificación dañada
MULTI_ENTIDADTablas desnormalizadas que agrupan entidades distintas
TIPOS_INCONSISTENTESFechas y monedas almacenadas bajo nomenclaturas heterogéneas
EXTRACCIONDocumentación no estructurada como PDFs planos y logs del sistema
ESQUEMAEstructura flotante que demanda inferencia tipológica activa
DATOS_LIMPIOSBaseline libre de imperfecciones para control de calidad

Formatos Soportados: CSV · Excel · JSON · JSONL · Log · XML · PDF · TOML

Flujo Lógico del Generador

  • LM Studio / OpenAI API: Genera dinámicamente contextos temáticos detallados vía `http://localhost:1234/v1`.
  • Banco de Fallback: Autonomía total mediante 140 temas precargados en local ante caídas de API.
  • Inyector de Caos: Incorpora deliberadamente 12 imperfecciones para degradar de forma estructurada los registros.

Comandos de Consola (chaos-generator)

# Generar todo con LM Studio activo
npm run generate
# Generar sin IA (temas predefinidos de fallback - mucho más rápido)
npm run generate:fast
# Generar únicamente el baseline de datos limpios
npm run generate:clean
# Control fino: 3 CSVs de REDUNDANCIA con 50 filas
npm run generate -- --category REDUNDANCIA --type csv --count 3 --rows 50
# Ejecución en modo simulado (Dry Run)
npm run generate:dry

02. Documentación Visual

HU-035: Diagramas interactivos, compilación automatizada y CI/CD de sincronización.

Diagramas como Código (Mermaid)

Arquitectura en Git

Los diagramas viven dentro de archivos Markdown tradicionales. Se editan mediante texto plano y son totalmente versionables.

10 Modelos Base

Abarca todos los flujos principales: procesamiento IA, base de datos relacional, flujos ETL, RBAC y regulaciones de privacidad.

diagram-exporter

Un transpilador inteligente que lee sintaxis Mermaid pura y la renderiza de forma óptima a 36 recursos de imagen.

Compilación Ágil de Diagramas

Instrucciones para la terminal del exporter:

# Exportar todos los diagramas del sistema
npm run diagrams
# Compilar diagramas específicos por su identificador
npm run diagrams:only -- 10
# Ver qué se generaría sin crear archivos
npm run diagrams:list

Sincronización Automática de docs/ (CI/CD)

  • Trigger: push en la rama main cuando hay cambios en docs/**.
  • Remoto destino: IA-DataFlow-Hub/docs (la acción usa actions/checkout@v4 apuntando al repo destino).
  • Autenticación: usa el secreto SYNC_PAT para hacer push al repo destino.
  • Flujo: checkout del repo origen y destino, copia de docs/ a destination/Docs/, commit y push si hay cambios.

Workflow: .github/workflows/sync-docs-to-docrepo.yml

Uso típico: Simplemente haga git push a main con cambios en docs/; la Action se encargará de sincronizar automáticamente.

03. Interfaz hu-cli

Conexión directa y ágil entre Historias de Usuario locales y el Kanban de GitHub Projects.

Terminal Interactiva (hu-cli)

  • Sincronizador Directo: Transforma la edición de archivos Markdown en cambios sobre los tableros de GitHub Projects.
  • Protección de Estados: El comando `hu update` se restringe a Backlog/Ready para evitar corrupciones de tareas activas.
  • Dependencias: Construido de manera ágil con `Commander.js`, `Chalk` para salida coloreada, y `gh` CLI local.

04. DVC (Data Version Control)

Control de versiones para datos a gran escala, separando el código del almacenamiento de datasets.

Control de Versiones de Datos (DVC)

Separación de Intereses

Mantiene metadatos ligeros (archivos `.dvc`) en Git, mientras los binarios pesados de datos se gestionan y almacenan independientemente.

Replicabilidad

Cualquier desarrollador puede sincronizar y descargar los datasets exactos de un commit específico con un solo comando: dvc pull.

Auditoría Completa

Integración directa con el flujo de Git para permitir trazabilidad y reversiones en los conjuntos de datos de entrenamiento generados.

Comandos de Gestión de Datasets

# Descargar los datos referenciados por DVC desde el remoto configurado
dvc pull
# Comprobar el estado local vs remoto de los archivos de datos
dvc status
# Añadir nuevos archivos o cambios en datos al control de DVC
dvc add path/to/data
# Enviar cambios de datos al remoto configurado
dvc push

Nota Operativa: Ejecute dvc pull antes de correr pipelines dependientes de datos si no tiene acceso al remoto, para sincronizar los archivos referenciados en datasets.dvc.

Autenticación Remota (Google Drive)

  • Backend GDrive: Los datasets pesados se almacenan de forma segura y económica en un bucket de Google Drive.
  • Flujo Interactivo: Ideal para desarrolladores. dvc pull levanta automáticamente un prompt de navegador para autorizar la lectura vía OAuth2.
  • Flujo CI/CD: Configuración manual de credenciales (client_id / client_secret) o cuentas de servicio para entornos sin cabeza (headless).
# Opción 1: Autorización en navegador (Desarrollo)
dvc pull
# Opción 2: Configurar credenciales explícitas (CI)
dvc remote modify gdrive gdrive_client_id <ID>
